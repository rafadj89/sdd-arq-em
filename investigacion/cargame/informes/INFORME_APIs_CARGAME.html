1. Resumen Ejecutivo

Se realizó un análisis técnico de descubrimiento (discovery) sobre la plataforma CARGAME del Ministerio de Minas y Energía de Colombia, accesible en https://cargame.minenergia.gov.co. El objetivo es identificar las APIs disponibles para consultar si terceros están registrados como proveedores/operadores de estaciones de carga eléctrica, como insumo para la solicitud formal de acceso a APIs ante el dueño de la plataforma.

💡 Hallazgo Principal

Se descubrieron 14 endpoints API funcionales que permiten consultar la totalidad de los 194 operadores/proveedores registrados (incluyendo NIT), las 139 estaciones de carga activas y los catálogos geográficos completos de Colombia. Estas APIs ya están accesibles públicamente con un token JWT estático embebido en el frontend.

5. Hallazgos de Seguridad

Durante el análisis se identificaron 5 hallazgos de seguridad que se recomienda reportar al dueño de la plataforma como parte de la comunicación formal de solicitud de APIs.

CRÍTICA

Token JWT estático hardcodeado en el frontend

El token JWT usado para autenticación está embebido directamente en el bundle JavaScript del frontend Angular. Este token fue emitido el 16 de julio de 2020 y nunca expira. Cualquier persona que inspeccione el código fuente puede extraerlo.

Payload decodificado: {"id":0,"usuario":"powerbi","role":"intelligence","iat":1594849631}

Impacto: Acceso sin autenticación a todos los endpoints de la API por cualquier persona.

CRÍTICA

Hashes bcrypt de contraseñas expuestos vía API

El endpoint GET /crg/usuario retorna los 272 registros de usuarios incluyendo el campo Clave con el hash bcrypt de cada contraseña.

"Clave": "$2b$10$3FkOuBVgeb5lYVrehaxLDuJJejXgwLbKhiEHBmfVG7tNOruDx9Xvy"

Impacto: Aunque bcrypt es resistente, exponer hashes permite ataques de fuerza bruta offline y es una violación de buenas prácticas de seguridad.

ALTA

Datos personales expuestos sin autenticación real

Las APIs exponen nombres, cédulas (CC), NITs, correos electrónicos, celulares y direcciones de 272 usuarios y 194 operadores. Estos datos son accesibles con el token JWT estático público.

Impacto: Posible violación de la Ley 1581 de 2012 (Protección de Datos Personales de Colombia) y Decreto 1377 de 2013.

MEDIA

CORS permisivo — APIs accesibles desde cualquier origen

Las APIs responden a peticiones cross-origin sin restricciones. Cualquier sitio web en internet podría hacer llamadas a las APIs de CARGAME.

MEDIA

Sin rate limiting aparente

No se detectaron headers de limitación de tasa (X-RateLimit-*) en las respuestas HTTP. Las APIs podrían ser abusadas con solicitudes masivas sin restricción.

La Unidad de Planeación Minero-Energética (UPME), en ejercicio de sus funciones de planeación del sector energético colombiano y en el marco del desarrollo de la infraestructura de electromovilidad del país, solicita formalmente acceso programático (vía APIs) a la plataforma CARGAME (https://cargame.minenergia.gov.co) para los siguientes fines:


1. Objetivo Principal

Validar si terceros (personas jurídicas o naturales) se encuentran registrados como proveedores/operadores de estaciones de carga de energía eléctrica para vehículos en la plataforma CARGAME, como requisito para procesos de verificación y planeación energética a cargo de la UPME.


2. APIs Requeridas


# API Requerida Propósito Prioridad
1 Consulta de operadores por NIT Verificar si un NIT está registrado como operador de estaciones de carga Alta

3. Requerimientos Técnicos

  • Documentación técnica de APIs (preferiblemente OpenAPI/Swagger)
  • Credenciales de acceso dedicadas para la UPME (API Key o OAuth2)
  • Ambiente de pruebas (sandbox) para desarrollo e integración
  • Especificación de límites de uso (rate limiting, cuotas)
  • SLA de disponibilidad del servicio
  • Punto de contacto técnico para soporte en la integración

4. Contexto Regulatorio

Esta solicitud se enmarca en las funciones de la UPME establecidas en el Decreto 1258 de 2013 y las políticas de promoción de la movilidad eléctrica contenidas en la Ley 1964 de 2019 y el Decreto 191 de 2021 del Ministerio de Transporte, así como la Resolución CREG relacionada con tarifas de carga eléctrica.


5. Compromiso de Uso

La UPME se compromete a:

  • Usar los datos exclusivamente para los fines institucionales descritos
  • Cumplir con la Ley 1581 de 2012 de Protección de Datos Personales
  • No redistribuir los datos a terceros sin autorización
  • Respetar los límites de uso establecidos

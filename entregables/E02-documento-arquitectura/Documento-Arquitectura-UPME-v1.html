1. Visión Ejecutiva

1.1 Perspectiva CEO / Director UPME

💼 Director UPME — Visión Estratégica

La Plataforma de Interoperabilidad de Electromovilidad representa el compromiso de la UPME con la transición energética de Colombia. En cumplimiento de la Resolución 40559/2025 del Ministerio de Minas y Energía, esta plataforma habilitará que cualquier usuario de vehículo eléctrico pueda cargar en cualquier estación pública, independientemente del operador, con transparencia total en precios y disponibilidad en tiempo real.

Objetivos estratégicos:

  • Cumplir los plazos mandatorios regulatorios (información pública Abr 2026, Guía Técnica May 2026, Go-Live Nov 2026)
  • Posicionar a Colombia como referente latinoamericano en interoperabilidad de carga eléctrica
  • Garantizar soberanía de datos — toda la información reside en territorio colombiano (OCI Región Bogotá)
  • Construir un ecosistema abierto basado en el estándar internacional OCPI 2.2.1
  • Asegurar la supervisión y auditoría por parte de la SIC para protección del consumidor

1.2 Perspectiva CTO / Líder Tecnológico

🛠️ CTO — Dirección Técnica

La arquitectura se diseñó bajo principios de sistemas distribuidos de misión crítica para gobierno, priorizando: seguridad por diseño, observabilidad primero, API-first, zero-trust, e infraestructura como código. El sistema está preparado para manejar 21.6M+ transacciones/mes con latencia P99 < 100ms en ingesta.

Decisiones clave:

  • Arquitectura event-driven con CQRS para separar escritura (ingesta CPOs) de lectura (consulta pública)
  • Microservicios por bounded context (DDD) — 7 contextos acotados independientes
  • Stack dual: Go 1.22+ para alta concurrencia (ingesta), Python 3.12+ para validación de schemas OCPI
  • OCI como cloud primaria con Kubernetes (OKE), Autonomous DB, Streaming (Kafka API)
  • Data Lake Medallion (Bronze→Silver→Gold) con trazabilidad end-to-end para SIC
  • Equipo: 8 personas IT (2 Sr Backend, 2 Jr Backend, 1 Sr Frontend, 1 DevOps Sr, 1 DevOps Jr, 1 Arquitecto)

1.3 Perspectiva Arquitecto de Soluciones

🏧️ Arquitecto Senior — Diseño de Referencia

Este documento presenta la línea base de arquitectura utilizando el modelo C4 (Context, Container, Component, Code) combinado con Domain-Driven Design (DDD). Cada decisión está documentada como ADR (Architecture Decision Record) con alternativas evaluadas e impacto regulatorio.

Principios arquitectónicos: SOLID, DRY, KISS, 12-Factor App, Fail-fast/Fail-safe, Design for Failure, Observability-first, Security by Design, API-first, Zero-trust, IaC.

Patrones aplicados: Event Sourcing selectivo, CQRS, Anti-Corruption Layer, Circuit Breaker, Bulkhead, Saga, Idempotency Guard, Medallion Architecture, Published Language, Open Host Service.

6.2 Gobernanza de Datos

Data Quality Engine

  • Freshness: Datos < 5 min
  • Volume: ±2σ baseline
  • Schema: OCPI 2.2.1 compliance
  • Completeness: Campos obligatorios
  • Consistency: Histórico vs actual
  • Anomalies: Z-score + seasonal decomposition

Data Catalog (OCI)

  • Clasificación PII: Ley 1581/2012
  • Linaje: Origen → transformación → destino
  • Glossario: Términos OCPI + regulatorios
  • Retención: Políticas por capa y tipo

7. Arquitectura de Seguridad

7.1 Defensa en Profundidad

Experto en Seguridad — Estrategia Zero-Trust

La plataforma implementa defensa en profundidad con 6 capas de seguridad. Cada request de un CPO atraviesa WAF → IP Filtering → mTLS → JWT/DPoP → API Gateway (rate limiting) → microservicio. El principio es zero-trust: todo tráfico se autentica y autoriza, incluso dentro del cluster.

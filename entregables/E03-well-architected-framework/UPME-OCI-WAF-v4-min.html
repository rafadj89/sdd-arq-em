2. Arquitectura OCI — Componentes & Diseño

Visión: Infraestructura cloud-native sobre OCI. Servicios gestionados para reducir carga operativa. GitOps (ArgoCD) multi-ambiente. Diseño event-driven para alta concurrencia (+500 CPOs / 21.6M tx/mes) con escalamiento horizontal automático y DRP integrado desde día n.

2.1 Principios & Decisiones Arquitectónicas

Principios

  • Cloud-native: Microservicios en OKE, event-driven con Kafka
  • Managed first: OKE, Streaming, Functions, ATP managed por Oracle
  • Event-driven: OCI Streaming (Kafka API) desacople ingesta/procesamiento
  • CQRS + Event Sourcing: Separación escritura/lectura, audit trail
  • GitOps: ArgoCD declarativo multi-ambiente
  • Security by design: Service Mesh mTLS, WAF, Vault Default (software keys), Zero Trust
  • DR by design: Full Stack DR cross-region desde día n

ADRs (Architecture Decision Records)

  • ADR-001: Microservicios Go (ingesta) + Python (validación) en OKE
  • ADR-002: OCI Streaming (Kafka API) event-driven
  • ADR-003: OCI Functions serverless para webhooks y DLQ
  • ADR-004: OCI Service Mesh mTLS automático inter-servicio
  • ADR-005: AI Data Platform: Object Storage medallion (Bronze/Silver/Gold) + Data Flow + Data Catalog + AI Services
  • ADR-006: Full Stack DR cross-region (RPO<1h, RTO<30min)
  • ADR-007: ArgoCD nativo en nodo OKE, CI/CD GitOps multi-ambiente + OCIR
  • ADR-008: CQRS + Event Sourcing + DLQ resiliencia

Cluster tipo Enhanced con Managed Nodes E5 (VM.Standard.E5.Flex). Despliegue declarativo via Manifiestos YAML. Opción Enhanced habilitada: Kubernetes API audit logging, cluster add-ons management, workload identity.

Sizing recomendado por nodo (PROD): VM.Standard.E5.Flex — 2 OCPU, 16 GB RAM, 100 GB SSD (Boot Volume).
Autoscaling: Mínimo 2 nodos (HA), máximo 4 nodos. Total pico: 8 OCPU, 64 GB RAM.
Justificación: Con 2 nodos (4 OCPU, 32 GB) se cubre la carga sostenida. Con 3-4 nodos (6-8 OCPU, 48-64 GB) se absorben picos, HPA burst, y mantenimiento rolling update sin degradación.
Enhanced features: API audit logging, cluster add-ons, workload identity, Kubernetes dashboard nativo.

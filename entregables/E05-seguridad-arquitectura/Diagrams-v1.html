Ecosistema OCPI 2.2.1 - Plataforma de Electromovilidad
Gobierno de Colombia

El Desafío de Interoperabilidad

La integración del estándar OCPI 2.2.1 transforma al Estado en el nodo central (Hub) de comunicaciones para el sector de recarga vehicular.

Esta centralización exige la implementación inmediata de una arquitectura Zero Trust (Confianza Cero) para orquestar la comunicación M2M (Máquina a Máquina).

Se requiere un filtrado estricto L3/L4, protección WAF avanzada contra inyecciones y validación rigurosa de tokens, garantizando que solo Proveedores (CPOs) autenticados consuman recursos del clúster.

WAF-->>CPO: 200 OK

Pilares de la Capa de Seguridad

1. IP Whitelisting (L4)

Bloqueo físico en el perímetro. Se descartan las peticiones TCP que no provengan de las IPs estáticas declaradas por cada CPO, ahorrando procesamiento.

2. Identidad Efímera

Sustitución de las viejas API Keys de OCPI por tokens JWT rotativos vía OAuth 2.1 gestionados por Keycloak, reduciendo riesgo por fugas.

3. Inspección en Proxy

Protección WAF profunda embebida directamente en el API Gateway. Previene inyecciones SQL y ataques malformados sin latencia perceptible.
